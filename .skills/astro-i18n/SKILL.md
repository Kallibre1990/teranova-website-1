# Teranova — Astro i18n

## Routing
- Configured in `astro.config.mjs` → `i18n`: `defaultLocale: 'ru'`, 9 `locales`, `routing.prefixDefaultLocale: false`.
- **9 languages.** Indexed/verified: **ru** (at `/`), **en** (`/en/`), **ko** (`/ko/`).
  Machine-translated, **noindex** (pending native review): **zh, ja, it, de, fr, tr**.
- Non-default locales are generated by dynamic routes `src/pages/[lang]/…` via `getStaticPaths`;
  ru pages live at the root (`index.astro`, etc.). `trailingSlash: 'always'`.
- Verified locales come from `verifiedLocales` in `src/i18n/ui.ts`. `BaseHead` emits
  hreflang **only for verified locales** and adds `noindex` on unverified ones;
  the sitemap includes verified locales only.

## Strings — never hard-code in markup
- All copy lives in `src/i18n/ui.ts` as nested objects per language.
- `ru` is the **reference shape**; `en` and `ko` are typed `: typeof ru`, so TypeScript fails the build if a key is missing in any language. Keep all three in sync.
- In a component: `import { ui } from '../i18n/ui'; const t = ui[lang];` then `t.hero.title_metal`, etc.

## Helpers — `src/i18n/utils.ts`
- `getLangFromUrl(url)` → active lang.
- `useTranslations(lang)` → dictionary.
- `localizePath('/catalog', lang)` → `/en/catalog/` etc. (trailing slash). Always build internal links with this.
- `anchorHref('ondemand', lang)` → localized home + `#anchor`.
- `getAlternates(path)` → hreflang list (used by `BaseHead`).

## hreflang + canonical
- Handled centrally in `src/components/BaseHead.astro`: self-canonical per locale, `<link rel="alternate" hreflang>` for ru/en/ko + `x-default` → ru.
- `@astrojs/sitemap` is configured with the same locales and emits hreflang in the sitemap. noindex utility pages (`/thanks`) are filtered out.

## Adding a page
1. Create `src/pages/<page>.astro` (ru), `src/pages/en/<page>.astro`, `src/pages/ko/<page>.astro`.
2. Add its strings + `meta` title/description to all three langs in `ui.ts`.
3. Use `<Base lang path title description>` — hreflang/canonical/sitemap come for free.

## Adding a language (future: zh, ja, it, de, fr, tr)
- Add to `locales` in `astro.config.mjs` and the sitemap `i18n.locales`.
- Add the language to `languages` in `ui.ts` and provide a full dictionary.
- **Policy:** publish a language **only after a native speaker proofreads it.**
  Do **not** ship machine translation. Until then, do not add it to `locales`.
